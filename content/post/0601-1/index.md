## 前言

在笔者看来，HiFi 市场的**高度信息不对等**导致了消费主义玄学和片面伪科学盛行，不仅阻碍行业良性发展，更造成了玩家挑选设备困难，最终催生无休止的“**以价论声**”焦虑。此背景下，唯有提升科学认知，建立可验证的判断框架，身为局中人的我们才能在鱼龙混杂的产品叙事中保持清醒，少交“智商税“。

然而科学 HiFi 的门槛很高————既要学习信号系统、模电数电、物理&心理声学，又要仪器测量、实践验证，若非相关专业，需要投入大量精力。

好在 AI 的迅速发展使得往高墙内一窥成为可能，本文所**云**便是通过大量 AI 问答（ Kimi 2.7 ）总结的，希望对读者有所启迪。

## HiFi 系统漫游

音频文件 → 解码（decode）→ PCM/DSD 数据流 → (DSP 处理) → 传输协议+物理层 → [DAC] → 模拟信号 → 音量控制（衰减）→ 电压放大 → 电流放大 → [耳机] → 人耳 → [人脑] → 听觉

不难看出从内存中的音频文件到人脑产生的听觉，其中经过了 1 次衰减、2 次放大、3 次转换（数模转换、电声转换、声-感知转换），这几个步骤在整个路径中对声音影响最大。

> 常见的 HiFi 产品都可以被拆分为这些环节，值得注意的是我们常说的“解码器”其实是一个源于早期翻译的**错误**称呼。DAC 全称 **Digital to Analog Converter** 顾名思义功能是数模转换，而解码（decode）的概念和编码（encode）是对应的。

## 音源的演进

### 1. 信号表示方式

#### 模拟音源（Analog）
**物理量连续变化**——声音以连续的物理波形形式记录和重放。

- **记录原理**：声波的振幅和频率直接对应记录介质上的物理变化（如黑胶沟槽的深浅/宽窄、磁带磁粉的磁化强度）
- **数学本质**：连续函数 $f(t)$，在任意时间点都有定义
- **特点**：信息密度"无限"（理论上），但每一次复制/传输都会引入噪声和失真（信噪比逐级劣化）

#### 数字音源（Digital）
**离散采样+量化**——将声波转换为一系列数字编码。

- **记录原理**：按固定时间间隔（采样率）测量声波振幅，并用有限位数的数字表示（量化）
- **数学本质**：离散序列 $f[n]$，仅在采样点有定义
- **核心参数**：
  - **采样率**（如44.1kHz、96kHz）：决定可还原的最高频率（奈奎斯特极限：$f_{max} = f_s/2$）
  - **位深**（如16bit、24bit）：决定动态范围（~6dB/bit，16bit≈96dB）
- **特点**：复制无损，但存在量化误差；需要重建滤波器将离散信号恢复为连续波形

### 2. 产品形态详解

#### 黑胶唱片

| 维度 | 科学细节 |
|------|---------|
| **记录方式** | 沟槽的左右摆动（立体声）/上下深度变化（单声道 legacy）直接对应声波振幅 |
| **物理限制** | 低频需要大振幅→沟槽变宽→唱片容量减少；高频受唱针质量/循迹能力限制（一般有效上限15-20kHz） |
| **失真来源** | 唱针摩擦沟槽（磨损）、偏芯/抖晃（wow & flutter）、共鸣、静电噪声 |
| **"温暖感"来源** | 谐波失真（尤其是偶次谐波）、低频共振、高频衰减形成的特定频响曲线 |

**科学事实**：黑胶的**本底噪声**（~-60dB）和**动态范围**（~60-70dB）远低于CD。其"好听"更多源于失真特性而非保真度。

#### CD

| 维度 | 科学细节 |
|------|---------|
| **标准参数** | 44.1kHz/16bit（1982年Sony/Philips红皮书标准） |
| **物理载体** | 聚碳酸酯基盘上的凹坑（pit）和平地（land），激光读取 |
| **容量** | 74分钟（源于贝多芬第九交响曲时长） |
| **纠错机制** | CIRC（交叉交织里德-所罗门码），可修复轻微划痕 |
| **理论性能** | 动态范围~96dB，频响0-22.05kHz |

**关键设计决策**：
- 44.1kHz采样率：确保20kHz音频带宽+过渡带保护，兼容视频同步
- 16bit：当时DAC技术限制与存储容量的平衡

#### 现代数字音频系统

当代数字音频已分化出多条技术路径：

##### A. 基于电脑的音频（PC-Fi）

```
电脑（USB/光纤/同轴输出）→ 外置DAC → 功放/有源音箱
```

**技术要点**：
- **USB音频**：异步传输模式（DAC主导时钟，减少抖动）
- **文件格式**：PCM（WAV/FLAC/AIFF）、DSD（1bit高频采样，如2.8MHz/5.6MHz）
- **位流 vs 包传输**：USB Audio Class 2.0支持高达PCM 768kHz/32bit或DSD 512
- **抖动（Jitter）**：数字时钟精度影响时域误差，现代异步方案已将其降至可忽略水平

##### B. 网络播放器/数播（Network Streamer）

**架构**：
```
网络（NAS/流媒体服务）→ 数播（内置CPU+解码）→ DAC/功放
```

**科学优势**：
- **电气隔离**：网络传输（TCP/IP）天然隔离前端噪声，优于USB直连
- **专用系统**：精简OS（如Volumio、Roon）减少电脑的多任务干扰
- **时钟精度**：高端数播采用飞秒级晶振（femtosecond clock，误差<1皮秒）

**流媒体协议**：
- UPnP/DLNA、AirPlay 2（ALAC编码）、Roon RAAT（专有低延迟协议）
- **MQA**（争议技术）：折叠高频信息至较低采样率容器，声称"无损"但涉及专利与认证争议

### 3. 关键科学对比

| 指标 | 黑胶 | CD | 高清数字（PCM 192/24） |
|------|------|-----|----------------------|
| **动态范围** | ~65dB | ~96dB | ~144dB |
| **频响** | 30Hz-15kHz（±3dB） | 0-20kHz（±0.5dB） | 0-96kHz |
| **失真（THD）** | 1-2% | <0.01% | <0.001% |
| **分离度** | ~30dB | >90dB | >120dB |
| **时基抖动** | 高（机械系统） | 低 | 极低（飞秒级） |
| **噪声** | 炒豆声、转盘噪声 | 量化噪声（dither处理） | 接近理论极限 |

### 4. 常见误区澄清

**"模拟更自然"**：模拟系统的非线性失真确实产生人耳偏好的偶次谐波，但这是**失真**而非**准确**

**"CD音质差"**：16/44.1经正确重建滤波后，在可听频段内已远超人类听觉极限（20kHz以上、120dB动态范围）

**"高采样率可听"**：96kHz+主要利于后期处理（防止滤波器混叠），而非人耳直接感知；DSD的高频噪声需陡峭低通滤波

**"数字冷、模拟暖"**：本质是频响和失真差异，可通过数字EQ模拟黑胶曲线（RIAA曲线本身就是标准化均衡）

待施工

黑胶是模拟载体，优点是无须数模转换，直接放大即可，自然度最佳。但信噪比等指标低，素质上限不高。
CD 是数字载体，其优势不在介质本身，而在于 CD 机到 DAC 的路径最短，受干扰最少。

数字音乐（本地或流媒体）只要是同一个文件，音质就没有区别。听出差异通常来自软件层面：是否独占输出、内部 DSP 等。
文件格式方面，PCM 与 DSD 是两大流派，前者多 bit，后者 1bit。PCM 原始格式为 .wav，.flac 与 .m4a 是无损压缩。认为能听出 wav 与 flac 区别是荒谬的——无损压缩是可逆的，编解码后的哈希值一致。若你的系统能听出区别，说明系统保真度堪忧。

> .m4a 多来自 Apple Music，因 Apple 默认施加 digital master 处理，源文件已不同，故声音不同——但这与 alac 编码无关。

> 有损格式显然不 HiFi。蓝牙协议的评价有三点：延迟是红线，稳定性是前提，码率与精度排在最后。 ldac 优于 aac 往往是以牺牲延迟和稳定性换取码率，且部分手机会内置升频。同延迟下，ldac 仍可能优于 aac。

PC-HiFi 的两大痼疾：

1. Windows 音频 API 缺陷大，常需独立 ASIO 驱动。
2. CPU 高频噪声经 USB 地线耦合进解码耳放。

网播通过无线网隔离电脑与解码耳放，解决了上述问题，但可能引入延迟。
数播则是将一台小"PC"内置于音频路径前端，缩短整体链路。

数字音频传输需关注三点： **误码率（现代已无问题）**、**时钟信号**、**电磁干扰**。

USB 异步传输下，时钟由 DAC 端主导。但 USB 的 GND 持续供电，会引入电脑端电磁干扰。评价 USB 唯二标准：连接是否紧密稳固，屏蔽是否过关。
网线、交换机理论上对音质无影响。

接地问题： 电脑外壳静电经 USB 导入解码端，会导致低频纯净度下降。

USB 传输原理：差分信号传递模拟电压，以阈值判 0/1，误码率可忽略。（详见 raintain.com）
